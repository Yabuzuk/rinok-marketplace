# 🧪 ТЕСТОВЫЙ РЕЖИМ ОПЛАТЫ

## 🎯 Проблема
Невозможно тестировать дальше без реальной оплаты.

## ✅ Решение
Добавлен тестовый режим - кнопки оплаты имитируют успешную оплату.

---

## 🔧 КАК ВКЛЮЧИТЬ

### Локально (разработка):
1. Файл `.env.local` уже создан с `REACT_APP_TEST_MODE=true`
2. Запустить: `npm start`
3. Все кнопки оплаты будут работать в тестовом режиме

### На продакшене (Firebase):
Добавить в консоли Firebase переменную окружения или пересобрать с флагом.

---

## 🎮 КАК РАБОТАЕТ

При нажатии кнопки "💳 Оплатить":
- **Тестовый режим ON**: Статус сразу меняется на `paid` или `products_paid`
- **Тестовый режим OFF**: Открывается форма Tinkoff

Алерт показывает: `✅ [ТЕСТ] Оплата успешна!`

---

## 🔄 КАК ВЕРНУТЬ ОБРАТНО

### Вариант 1: Удалить файл
```bash
del .env.local
```

### Вариант 2: Изменить значение
В `.env.local` изменить на:
```
REACT_APP_TEST_MODE=false
```

### Вариант 3: Git откат
```bash
git checkout HEAD -- src/pages/CustomerDashboard.tsx
```

---

## 📝 ЧТО ИЗМЕНЕНО

**Файл:** `src/pages/CustomerDashboard.tsx`

**Добавлено:**
```javascript
const isTestMode = process.env.REACT_APP_TEST_MODE === 'true';

if (isTestMode) {
  // Имитируем успешную оплату
  await onUpdateOrder(order.id, {
    status: newStatus,
    paidAt: new Date().toISOString()
  });
  alert('✅ [ТЕСТ] Оплата успешна!');
  return;
}
```

---

## 🧪 ТЕСТИРОВАНИЕ

1. Создать заказ
2. Менеджер подтверждает
3. Нажать "Оплатить товары" → статус `products_paid`
4. Менеджер рассчитывает доставку
5. Нажать "Оплатить доставку" → статус `paid`
6. Менеджер собирает → `ready`
7. Менеджер отправляет → `delivering`

Все без реальной оплаты! 🎉

---

## ⚠️ ВАЖНО

- Тестовый режим НЕ деплоится на продакшен (`.env.local` в `.gitignore`)
- Для продакшена нужно добавить переменную в Firebase Console
- Или пересобрать с флагом: `REACT_APP_TEST_MODE=true npm run build`
