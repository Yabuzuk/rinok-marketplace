# Исправление товаров после миграции

## Проблема
После миграции с Supabase на Firebase некоторые товары отображаются в интерфейсе, но физически не существуют в Firebase Firestore. Это приводит к ошибке "No document to update" при попытке редактирования.

## Решение

### Вариант 1: Автоматическое исправление через браузер

1. Откройте файл `fix-products.html` в браузере
2. Нажмите кнопку "Запустить исправление"
3. Скрипт автоматически:
   - Найдет все товары в Firebase
   - Проверит их существование
   - Восстановит проблемные товары с новыми ID

### Вариант 2: Ручное исправление

1. Зайдите в Firebase Console: https://console.firebase.google.com/project/kalaktika-app/firestore
2. Откройте коллекцию `products`
3. Найдите товары с проблемными ID (например, `1767097143865`)
4. Удалите их из Firestore
5. В приложении нажмите "Обновить" в кабинете продавца

### Вариант 3: Через код приложения

Исправления уже внесены в код:
- При ошибке "товар не найден" он автоматически удаляется из локального состояния
- Происходит перезагрузка данных с сервера
- Модальное окно редактирования закрывается

## Проверка результата

После исправления:
1. Все товары должны корректно редактироваться
2. Ошибки "No document to update" исчезнут
3. Товары сохранятся, но получат новые Firebase ID

## Деплой исправлений

```bash
npm run build
firebase deploy --only hosting
```

Исправления уже внесены в код и готовы к деплою.
