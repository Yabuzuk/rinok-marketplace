# Этап 10 - Уведомления ✅

## Реализованные уведомления

### 1. Групповые заказы

#### Участник присоединился
- **Кому**: Организатору группового заказа
- **Когда**: При вызове handleJoinGroupOrder()
- **Сообщение**: "{Имя} присоединился к вашему совместному заказу {КОД}"

#### Все оплатили товары
- **Кому**: Менеджеру
- **Когда**: Последний участник оплачивает товары
- **Сообщение**: "Все участники оплатили товары в заказе {КОД}"

#### Все оплатили доставку
- **Кому**: Менеджеру
- **Когда**: Последний участник оплачивает доставку
- **Сообщение**: "Все участники оплатили доставку в заказе {КОД}. Можно отправлять!"

### 2. Пулы доставки

#### Пул закрывается через 30 минут
- **Кому**: Всем участникам пула
- **Когда**: За 30 минут до closeTime
- **Сообщение**: "⚠️ Пул доставки закроется через 30 минут! Успейте оформить заказ"
- **Отправляется**: Один раз (отслеживается в warningsSent)

#### Пул закрыт
- **Кому**: Всем участникам пула
- **Когда**: При автоматическом закрытии пула
- **Сообщение**: "Пул доставки закрыт. Менеджер скоро рассчитает стоимость доставки"

#### Доставка рассчитана
- **Кому**: Всем участникам пула
- **Когда**: Менеджер выставляет счета за доставку
- **Сообщение**: "📦 Доставка рассчитана! Оплатите в течение 25 минут"

### 3. Оплата доставки

#### Напоминание об оплате (за 5 минут)
- **Кому**: Покупателю с неоплаченной доставкой
- **Когда**: За 5 минут до истечения дедлайна
- **Сообщение**: "⏰ Осталось 5 минут для оплаты доставки заказа №{НОМЕР}!"
- **Отправляется**: Один раз (отслеживается в paymentWarningsSent)

#### Участник исключен
- **Кому**: Исключенному участнику
- **Когда**: Истек таймер оплаты доставки
- **Сообщение**: "Вы исключены из совместного заказа {КОД} за неоплату доставки. Средства за товары будут возвращены"

#### Доставка пересчитана
- **Кому**: Оставшимся участникам
- **Когда**: После исключения неплательщика
- **Сообщение**: "Доставка пересчитана для заказа {КОД}. Проверьте новую сумму"

### 4. Статусы заказов

#### Заказ собран
- **Кому**: Покупателю
- **Когда**: Статус меняется на 'ready'
- **Сообщение**: "📦 Ваш заказ №{НОМЕР} собран и готов к отправке"

#### Заказ в пути
- **Кому**: Покупателю
- **Когда**: Статус меняется на 'delivering'
- **Сообщение**: "🚚 Заказ №{НОМЕР} передан в доставку. Скоро будет у вас!"

#### Заказ доставлен
- **Кому**: Покупателю
- **Когда**: Статус меняется на 'delivered'
- **Сообщение**: "✅ Заказ №{НОМЕР} доставлен! Приятного аппетита!"

### 5. Существующие уведомления (из предыдущих этапов)

- Новый заказ → Продавцу
- Подтверждение заказа → Покупателю
- Оплата товаров → Продавцу и менеджеру
- Изменение заказа → Покупателю

## Технические детали

### Отслеживание отправленных уведомлений
```typescript
let warningsSent: Set<string> = new Set(); // Для пулов
let paymentWarningsSent: Set<string> = new Set(); // Для оплаты
```

### Очистка после события
- После закрытия пула: `warningsSent.delete(poolId)`
- После истечения таймера: `paymentWarningsSent.delete(orderId)`

### Интервалы проверки
- **30 минут до закрытия**: `timeDiff <= 30 * 60 * 1000`
- **5 минут до оплаты**: `timeDiff <= 5 * 60 * 1000`

### Условия отправки
```typescript
if (timeDiff > 0 && timeDiff <= threshold && !sent.has(id)) {
  // Отправляем уведомление
  sent.add(id);
}
```
