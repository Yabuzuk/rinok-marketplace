# Тест мульти-павильонных заказов

## Что изменилось:

### 1. Типы (types/index.ts)
- Добавлено поле `parentOrderId?: string` для группировки связанных заказов

### 2. Логика создания заказов (App.tsx)
- При оформлении заказа товары группируются по `pavilionNumber`
- Если товары из одного павильона → создается один заказ
- Если товары из разных павильонов → создается отдельный заказ для каждого павильона
- Все связанные заказы получают одинаковый `parentOrderId`
- Каждый продавец получает уведомление только о своем заказе

### 3. Интерфейс заказов (OrdersPage.tsx)
- Заказы группируются по `parentOrderId`
- Мульти-павильонные заказы показываются как "Заказ из N павильонов"
- Каждый павильон отображается отдельно с кнопкой оплаты
- Покупатель видит отдельные СБП ссылки для каждого павильона

### 4. Корзина (Cart.tsx)
- Уже корректно группирует товары по павильонам при создании заказа

## Как тестировать:

1. **Войти как покупатель**
2. **Добавить товары из разных павильонов в корзину**
3. **Оформить заказ** → должно создаться несколько заказов
4. **Проверить страницу "Мои заказы"** → должна показать группировку
5. **Проверить оплату** → должны быть отдельные кнопки для каждого павильона
6. **Войти как продавец** → должен видеть только свои заказы
7. **Проверить уведомления** → каждый продавец получает уведомление только о своем заказе

## Результат:
- ✅ Каждый продавец получает только свои заказы
- ✅ Каждый продавец получает оплату только за свои товары  
- ✅ Покупатель видит удобную группировку связанных заказов
- ✅ Отдельная оплата для каждого павильона через СБП
- ✅ Корректные уведомления для всех участников
