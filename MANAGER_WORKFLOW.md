# 🎯 РАБОТА МЕНЕДЖЕРА С РАЗНЫМИ ТИПАМИ ЗАКАЗОВ

## Три типа заказов:

1. **Индивидуальный заказ** (`deliveryType: 'individual'`)
2. **Автоматическая группировка** (`deliveryType: 'auto_group'`)
3. **Совместный заказ соседей** (`deliveryType: 'neighbor_group'`)

---

## 📋 FLOW ДЛЯ КАЖДОГО ТИПА

### 1️⃣ ИНДИВИДУАЛЬНЫЙ ЗАКАЗ

**Покупатель:**
1. Оформляет заказ
2. Выбирает "Индивидуальная доставка"
3. Заказ создается со статусом `pending`

**Менеджер:**
1. Видит заказ в списке "Новые заказы"
2. Проверяет наличие товара
3. Подтверждает заказ (статус → `confirmed`)
4. Вводит стоимость доставки (например, 500₽)
5. Нажимает "Добавить доставку"
6. Заказ → `payment_pending`
7. После оплаты покупателем → собирает → отправляет

---

### 2️⃣ АВТОМАТИЧЕСКАЯ ГРУППИРОВКА

**Покупатель:**
1. Оформляет заказ
2. Выбирает "Групповая доставка (ждать формирования)"
3. Выбирает дату (сегодня/завтра/послезавтра)
4. Выбирает временной слот (11-13, 13-15, 15-17)
5. Сразу оплачивает товары (без доставки)
6. Заказ создается со статусом `products_paid`

**Менеджер:**

1. Видит заказ в пуле доставки (например, пул 13:00-15:00, завтра)

2. В 14:30 пул автоматически закрывается

3. Менеджер видит все заказы в этом пуле:
   - Заказ #1234: ул. Ленина, 15 - 800₽
   - Заказ #1235: ул. Ленина, 20 - 600₽
   - Заказ #1236: ул. Мира, 8 - 1200₽

4. Менеджер группирует заказы по адресам/районам:
   - Группа А (ул. Ленина): #1234 + #1235
   - Группа Б (ул. Мира): #1236

5. Рассчитывает доставку через Яндекс API:
   - Группа А: 400₽ на двоих
   - Группа Б: 500₽ один

6. Распределяет доставку:
   - Заказ #1234: 400₽ × (800/1400) = 229₽
   - Заказ #1235: 400₽ × (600/1400) = 171₽
   - Заказ #1236: 500₽

7. Нажимает "Выставить счета за доставку"

8. Покупатели получают уведомления и оплачивают доставку (25 минут)

9. После оплаты всех → собирает → отправляет группой

---

### 3️⃣ СОВМЕСТНЫЙ ЗАКАЗ СОСЕДЕЙ

**Покупатели:**
1. Организатор создает совместный заказ с кодом
2. Соседи присоединяются по коду
3. Выбирают дату и временной слот
4. Каждый оплачивает свои товары до закрытия пула
5. Создается один GroupOrder с несколькими участниками

**Менеджер:**

1. Видит совместный заказ в пуле (например, SOSEDI-A7K9)

2. В 14:30 пул закрывается

3. Менеджер видит детали:
   ```
   📦 Совместный заказ SOSEDI-A7K9
   📍 Адрес: ул. Ленина, 15, подъезд 2
   👥 Участники (3):
      • Иван И. - 800₽ ✅ оплачено
      • Мария П. - 700₽ ✅ оплачено
      • Петр С. - 500₽ ✅ оплачено
   💰 Итого товары: 2000₽ (все оплачено)
   ```

4. Вводит стоимость доставки: 500₽

5. Система автоматически распределяет:
   - Иван И.: 200₽ (40%)
   - Мария П.: 175₽ (35%)
   - Петр С.: 125₽ (25%)

6. Нажимает "Выставить счета за доставку"

7. Все участники получают уведомления (25 минут на оплату)

8. После оплаты всех → собирает → отправляет одной посылкой

## 🔄 КЛЮЧЕВЫЕ ОТЛИЧИЯ ДЛЯ МЕНЕДЖЕРА

| Тип заказа | Когда появляется | Что делает менеджер | Когда выставляет доставку |
|------------|------------------|---------------------|---------------------------|
| **Индивидуальный** | Сразу после оформления | Подтверждает (confirmed) + добавляет доставку сразу | Сразу при подтверждении |
| **Автогруппировка** | После оплаты товаров покупателем | Группирует с другими по адресу/району | После закрытия пула (14:30) |
| **Совместный** | После оплаты товаров всеми участниками | Вводит одну стоимость на всех | После закрытия пула (14:30) |

## 📅 ВРЕМЕННЫЕ СЛОТЫ И ДАТЫ

### Даты доставки:
- 📅 **Сегодня** - доставка в текущий день
- 📅 **Завтра** - доставка на следующий день
- 📅 **Послезавтра** - доставка через 2 дня

### Временные слоты:
- ⏰ **11:00-13:00** - закрытие в 12:30
- ⏰ **13:00-15:00** - закрытие в 14:30
- ⏰ **15:00-17:00** - закрытие в 16:30

Пул закрывается автоматически за **30 минут** до конца слота.

## ✅ ИТОГОВЫЙ АЛГОРИТМ ДЛЯ МЕНЕДЖЕРА

### Утром (работа с новыми заказами):
1. Обрабатывает **индивидуальные заказы** - подтверждает (confirmed) + добавляет доставку сразу
2. Видит **автогруппировку** и **совместные заказы** в пулах - ждет закрытия

### В момент закрытия пула (12:30, 14:30, 16:30):
1. Пул автоматически закрывается
2. Менеджер видит все заказы пула
3. Группирует автогруппировку по адресам (вручную или автоматически)
4. Для каждой группы вводит стоимость доставки
5. Система распределяет доли
6. Выставляет счета всем

### После оплаты доставки:
1. Собирает все заказы группы
2. Отправляет одной доставкой

## 📝 ВАЖНЫЕ МОМЕНТЫ ДЛЯ РЕАЛИЗАЦИИ

### 1. Статусы заказов для разных типов:

**Индивидуальный:**
- `pending` → `confirmed` → `payment_pending` → `paid` → `ready` → `delivering` → `delivered`

**Автогруппировка:**
- `products_paid` → `delivery_calculated` → `paid` → `ready` → `delivering` → `delivered`

**Совместный:**
- `products_paid` → `delivery_calculated` → `paid` → `ready` → `delivering` → `delivered`

### 2. Структура payments в Order:

```typescript
payments: {
  products: {
    status: 'pending' | 'paid',
    amount: number,
    paidAt?: string
  },
  delivery: {
    status: 'pending' | 'paid',
    amount: number,
    paidAt?: string,
    calculatedAt?: string,
    paymentDeadline?: string // для таймера 25 минут
  }
}
```

### 3. Логика распределения доставки:

```typescript
// Для автогруппировки и совместных заказов
function calculateDeliveryShares(orders: Order[], totalDeliveryCost: number) {
  const totalProductsCost = orders.reduce((sum, order) => sum + order.productsTotal, 0);
  
  return orders.map(order => ({
    orderId: order.id,
    share: (order.productsTotal / totalProductsCost) * totalDeliveryCost,
    percentage: (order.productsTotal / totalProductsCost) * 100
  }));
}
```

### 4. Таймер оплаты доставки:

- **25 минут** на оплату после выставления счета
- Уведомление за **5 минут** до истечения
- При истечении: исключение + пересчет для остальных

1. ✅ **Сначала**: Индивидуальные заказы (простейший flow)
