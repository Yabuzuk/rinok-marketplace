## Логика работы

### Автоматическое закрытие пула:
1. Планировщик проверяет каждую минуту
2. Сравнивает текущее время с pool.closeTime
3. Если время пришло → вызывает handlePoolAutoClose()
4. Пул переводится в статус 'closed'
5. Всем участникам отправляются уведомления

### Истечение таймера оплаты:
1. При выставлении счетов устанавливается deadline (текущее время + 25 минут)
2. Планировщик проверяет каждую минуту
3. Если deadline истек и статус 'delivery_pending':
   - Находит неоплативших участников
   - Исключает их из группового заказа
   - Пересчитывает доли для оставшихся
   - Отправляет уведомления всем

### Пересчет долей доставки:
```typescript
const remainingParticipants = groupOrder.participants.filter(
  p => !unpaidParticipants.find(up => up.userId === p.userId)
);

const newShares = calculateDeliveryShares(
  remainingParticipants,
  groupOrder.deliveryPrice
```

## Технические детали

### Формат дедлайна:
```typescript
const deadline = new Date();
deadline.setMinutes(deadline.getMinutes() + 25);
order.deliveryPaymentDeadline = deadline.toISOString();
```

### Интервал проверки:
- 60000 мс (1 минута)
- Оптимально для баланса между актуальностью и нагрузкой

### Предупреждения:
- За 5 минут до истечения (shouldSendWarning)
- Можно использовать для напоминаний пользователям
